## 📊 Инструменты мониторинга

### 1. Health Check Script
Используйте скрипт `scripts/health-check.sh` для комплексной проверки системы:

```bash
cd /home/u2168272/arq
bash scripts/health-check.sh
```

**Что проверяется:**
- ✅ Использование дискового пространства (предупреждения при 70% и 85%)
- ✅ Статус PM2 процессов
- ✅ Доступность API сервера (https://arq-ai.ru/api/health)
- ✅ Логи подключения к базе данных
- ✅ Использование оперативной памяти
- ✅ Последние ошибки PM2

### 2. PM2 Monitoring

**Проверка статуса процессов:**
```bash
pm2 list
pm2 status
```

**Просмотр логов:**
```bash
# Все логи
pm2 logs

# Последние 50 строк
pm2 logs --lines 50

# Только ошибки
pm2 logs --err

# Логи конкретного приложения
pm2 logs arq-backend
```

**Мониторинг в реальном времени:**
```bash
pm2 monit
```

### 3. Проверка дискового пространства

```bash
# Общее использование

# Детальный анализ папок
du -sh /home/u2168272/arq/*

# Поиск больших файлов
find /home/u2168272 -type f -size +100M -exec ls -lh {} \;
```

## ⚠️ Пороговые значения и предупреждения

### Дисковое пространство
- **70-85%**: ⚡ Внимание - рекомендуется очистка
- **85%+**: ⚠️ Критично - требуется немедленная очистка
- **95%+**: 🚨 Опасно - система может перестать работать

### Память
- **<80%**: ✅ Нормально
- **80-90%**: ⚡ Повышенное использование
- **>90%**: ⚠️ Критично - возможны падения процессов

### PM2 Процессы
- **online**: ✅ Работает нормально
- **stopped**: ⚠️ Процесс остановлен - требуется рестарт
- **errored**: 🚨 Ошибка - проверьте логи
- **one-launch-status**: ⚠️ Процесс запускается один раз (возможны проблемы)

## 🚨 Действия при проблемах

### Проблема: Высокое использование диска

**Диагностика:**
```bash
du -sh /home/u2168272/arq/* | sort -h
du -sh /home/u2168272/arq/node_modules | sort -h
```

**Решение:**
```bash
# Очистка логов
pm2 flush

# Очистка кэша npm
npm cache clean --force

# Очистка node_modules (осторожно!)
cd /home/u2168272/arq
rm -rf node_modules
npm install --production
```

### Проблема: PM2 процесс не запускается

**Диагностика:**
```bash
pm2 logs arq-backend --lines 100
pm2 describe arq-backend
```

**Решение:**
```bash
# Перезапуск
pm2 restart arq-backend

# Полный рестарт с очисткой
pm2 delete arq-backend
npm run start:prod
```

### Проблема: Конфликт портов

**Диагностика:**
```bash
lsof -i :3000
netstat -tulpn | grep :3000
```

**Решение:**
```bash
# Остановить все процессы PM2
pm2 stop all
pm2 delete all

# Перезапустить
npm run start:prod
```

### Проблема: База данных недоступна

**Проверка:**
```bash
psql -h localhost -U arq_user -d arq_db -c "SELECT 1;"
```

**Если не подключается** - проверьте статус PostgreSQL или обратитесь к хостинг-провайдеру Beget.

## 📈 Метрики для отслеживания

### Производительность
- Response time API (<500ms нормально)
- Memory usage (<80% нормально)
- CPU usage (<70% нормально)

### Доступность
- Uptime API сервера (цель: >99%)
- Database connection success rate
- PM2 процессы в статусе "online"

### Ресурсы
- Disk space usage (<70% рекомендовано)
- Memory available (>1GB рекомендовано)
- PM2 restart count (низкий - хорошо)
